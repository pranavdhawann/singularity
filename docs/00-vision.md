# Vision

Future is a local assistant IDE for power users and developers who want one
assistant that understands their work over time.

The product should feel less like a chat tab and more like a command center:
searchable memory, project context, tasks, files, models, permissions, and
actions in one local interface.

## Problem

Current assistant tools forget too much, scatter context across many chats, and
hide important state behind provider-specific interfaces. Users repeatedly
re-explain who they are, what they are building, what decisions were made, and
what constraints matter.

Developer AI tools are powerful, but many focus on immediate coding actions
rather than long-term personal context. Personal assistant tools can have
memory, but many are chat-first or channel-first. Local AI tools can be private,
but often lack strong memory organization and action permissions.

## Thesis

The next useful assistant is not a bigger chat box. It is a local command center
with durable memory, visible permissions, model choice, and a timeline that
keeps the assistant relationship coherent across months of work.

## Target Users

Future should serve two overlapping audiences:

- Power users who want a personal operating layer across notes, files, tasks,
  contacts, chats, and workflows.
- Developers and builders who want an assistant that understands projects,
  repos, docs, terminals, design decisions, and implementation history.

Privacy-first behavior is a baseline expectation, not the whole pitch. The
headline is memory, control, and model freedom.

## Product Promise

Future should help users answer:

- What does my assistant know about me and my work?
- Where did that memory come from?
- Which model will handle this task?
- What is the assistant allowed to do right now?
- What changed, when, and why?
- What context was used to produce this answer or action?

## Long-Term Ambition

Future can become the local control plane for a user's AI tools:

- one memory system
- one timeline
- one permission system
- many models
- many tools
- many workspaces
- user-controlled automation
